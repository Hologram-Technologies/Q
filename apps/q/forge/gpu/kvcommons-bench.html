KV-commons · prefix-pin TTFT — BitNet-2B streamed from HuggingFace (serverless)

The thesis: a shared prefix (system prompt / RAG context / few-shot block) should be prefilled once, then reused — so TTFT depends only on the NEW query, not on the prefix length. COLD = today's behavior when a new conversation reuses the same system prompt: reset() then decode(prefix + query) re-prefills the whole prefix. WARM = the prefix is pinned (sync(prefix) once, then truncateTo(|prefix|) per query); only the query is prefilled. Gate = byte-identical first token COLD vs WARM (the reuse is sound, not approximate). Read window.__kv.
